Inputs and Outputs

The fundamental building block of tests is the “code unit”. What forms a unit can vary between domains and projects. Unit tests isolate behavior so you don't have to involve the whole system. i.e. things behind a login

Unit test qualities

  • Test in isolation 👍
  • Inputs: stubs/fixture data 👍
  • Often headless 👍
  • Fast changing of state 👍
In aggregate, unit tests provide broad and detailed coverage of the actual code base. Compare outputs from known inputs. Fake as much as possible outside the unit’s wall so that bugs in that code don’t affect the test. For a unit of code to actually be useful it has to take some inputs, do something, and then produce outputs.

Writing Testable Code

Code needs enough logic to test

When the logic behind the method is complex enough that you feel you need to test extensively to verify that it works. Code smell: when there's a bunch of business logic in your tests

Code under test
must be portable

Sometimes it's as simple as exporting a function in your development environment. Or, you might have to refactor the code to isolate it and pass in the inputs.

Tests shouldn’t require
one-to-one changes in code

Low return-on-investment is when tests get commented out. Excessive testing makes code harder to change. Code smell of testing implementation. React Fiber (React 16) was a successful example of this, by testing public APIs that had to be maintained for backcompat

Readable and compact

Snapshot Testing

Let's talk briefly about a different technique: Snapshot testing. It's all about testing a payload to see if it changed. It's good for regression tests, especially for public APIs or JSON output. But like any testing method, you want to make sure you're using it effectively and testing things of value

Visual Snapshot Testing

In a similar way, visual snapshot testing could help you with regressions of visual designs. It's probably only useful for pattern libraries where you'd want to know if something changed. Less useful in projects with a lot of visual churn
